# Sorted Collections Performance Test

Проект для сравнения производительности и демонстрации различий между `SortedList` и `SortedDictionary` в C#.

## Описание

Этот проект сравнивает две отсортированные коллекции в .NET:
- **SortedList** - использует два массива (ключи и значения), эффективен для памяти но медленнее при вставке
- **SortedDictionary** - использует красно-черное дерево, быстрее при вставке но использует больше памяти

## Особенности коллекций

### SortedList<TKey, TValue>
- Хранит данные в двух отсортированных массивах
- **Плюсы**: Эффективное использование памяти, быстрый доступ по индексу
- **Минусы**: Медленная вставка и удаление (O(n))
- **Лучше использовать**: Когда данные редко изменяются, но часто читаются

### SortedDictionary<TKey, TValue>
- Использует бинарное дерево поиска (красно-черное дерево)
- **Плюсы**: Быстрая вставка и удаление (O(log n))
- **Минусы**: Больше использует памяти, нет доступа по индексу
- **Лучше использовать**: Когда часто добавляются/удаляются элементы

## Требования

- .NET 6.0 или выше
- Компилятор C#
